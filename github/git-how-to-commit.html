Github Command Line for Beginners

With acknowledgements to the contributions of Aurora Walker & the official GitHub cheat sheet

  1. Have some kind of command line tool on your computer.
    1. Link for Mac?
    2. Link for PC?
  2. Start the console.
  3. Navigate to the directory where your project files live.
    1. cd folder // this will change the directory. For example: cd documents
    2. Use the tab key to fill in more information
  4. If this is the first time you are accessing a project you will need to copy the files to your computer.
    1. git clone clone url // e.g. /gitusername/my-first-project
  5. Once the files are on your computer you can look into the current status of the information.
    1. git status // lists all new or modified files you have that are ready to be committed
    2. git log //
      // use 'q' to exit
  6. If your files are ready to go you can tell github to take a 'snapshot' of them to get them ready to upload.
    1. There are two ways to add your files:
    1. git add . // add all files
    2. git add filename // add individual files
  7. Once the files are added, you are ready to commit your files permanently into the version history.
    1. git commit -m "your descriptive commit message"
      // e.g. git commit -m "adding a purple box"
  8. Depending on how much time has passed, you may want to pull down the latest updates from the server before committing.
    1. git pull --rebase
      // pulls the latest files from the server and incorporates any changes
  9. You are now ready to submit your files up to the repo!
    1. git checkout -b branch-name --track origin master
      // let's break this down:
      checkout = switches you to the specified branch directory and updates it, branch-name = defines the name of the branch you are working on, -b = causes a new branch to be created, and then checked out, -- track =?, : =?
  10. Push your changes.
    1. git push origin mybranch[oncomputer]:mybranch[onserver]
      // upload all local commits to github
